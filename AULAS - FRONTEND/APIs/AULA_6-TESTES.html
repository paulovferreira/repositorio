TESTES

Realizar testes é muito importante. Manter uma rotina automatizada de testes garante que não quebraremos o código por negligência ou descuido.

TIPOS DE TESTE

Todo teste se resume em:
ENTRADA - PROCESSAMENTO - SAÍDA.

TIPOS DE TESTE

TESTE UNITÁRIO

- Teste unitário serve para testar um módulo / função específica do meu arquivo.
- Teste para ser realizado apenas para testar funções (menor parte) de um projeto.
- TESTE UNITÁRIO SEMPRE TESTARÁ A RESPOSTA LÓGICA PARA A FUNÇÃO.

No REACT:
- testa componentes
- Renderização de componentes.

TESTE DE INTEGRAÇÃO

É o conjunto de testes unitários.
- Fará o teste de resultado de ação.
- O MELHOR TESTE, pois testa a integração entre funções, componentes, elementos e unidades.

TESTE E2E - TESTE COMPLETO DE USABILIDADE

É o mais abrangente de todos.
- Cria rotinas aplicadas no navegador.
- É como um conjunto de testes de integração.
- Ele realiza os testes no próprio navegador.
- Simula uma ação específica executada pelo usuário.
- Basicamente um teste de fluxo de acontecimento.

O QUE DEVE-SE TESTAR?

Obviamente não desenvolvemos rotina de testes para todas as funções, se não passariamos mais tempo programando testes do que funcionalidade.

É preciso definir prioridades para testar:
- Definir o CORE do seu projeto.
- Testar principalmente áreas que serão manipuladas pelo usuário.
- Testar áreas onde existem integrações externas.
- Páginas que tem reuso de componentes críticos.
- Carregamentos de dados da base (sobrecarga)

IMPORTANTE

Não tome mais tempo planejando testes do que desenvolvendo.
Testes são um auxílio e não a prioridade do sistema.
