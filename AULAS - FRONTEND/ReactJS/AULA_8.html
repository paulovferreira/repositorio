ROUTES - GRUPOS DE ROTAS.

Podemos criar grupos de rotas aninhadas sobre o mes prefixo do seu sistema.


Página Home = "/"
Página Sobre = "/Sobre"
Página Sobre - Paulo = "/Sobre/Paulo"
Página Sobre - Alunos = "/Sobre/Alunos"

Perceba que a medida que vamos precisando aninhar as páginas, podemos criar "sublibnks" para cada uma delas


IMPORTANTE

Para cada nova rota em uma nova página, precisamos do IMPORT específico da página de referência.

Criando Rota para 404 - Tratamento de Erro

Caso o usuário informe uma página inexistente, seja via link ou seja via aplicação externa, precisamos estar preparados para tratar essa excessão.


PARA ISSO, É NECESSÁRIO CRIAR UMA PÁGINA CHAMADA "NOTFOUND.TSX"

Essa será a página a ser exibida quando um link inexistente for informado em seu projeto.

CRIANDO A ROTA NOTFOUND

A rota para as páginas não encontradas, será uma rota generica. Dessa forma, criamos um "coringa" para caso nenhuma das rotas sejam encontradas, esse meu elemento (NOTFOUND) seja exibido

< Route path="*" element={< NotFound />} />
Obs. Lembrando que para isso, precisaremos do import do componente NOTFOUND.

LINKS

Essa é a caracteristica PRINCIPAL da biblioteca Router.
EXIBIR ELEMENTOS SEM RECARREGAR A TELA.

A Partir dos LINKS o sistema processará todo o HTML já armazenado para exibir na tela sem a necessidade de reprocessamento total.

Utilizando Links

Os links utilizando ROUTERS server para efetivamente criar a interação entre as rotas já criadas.
Para conseguir vizualizar melhor, vamos garantir a seguinte estrutura:

1) Criar o componente Cabecalho.
2) Criar o componenet MENU.
3) Incluir o componente Menu dentro do componente Cabecalho.
4) Icluir o componente cabeçalho dentro da sua página HOME.

Agora, vamos criar o links para a página utilizando os menus criados.


Importando o elememento LINK
import {Link} from 'react-router-dom';


Essa importação garantirá que possamos utilizar os Links do Router.

Utilizando o Link < ul>
< li> < Link to="/sobrealuno"> Sobre Paulo < /Link> < /li>
< li> < Link to="/sobreprofessor"> Sobre Alunos < /Link> < /li>
< li> < Link to="/sobredirecao"> Sobre Alunos < /Link> < /li>
< /ul>

Diferença entre Link e Ancoras.

Ancoras
< a href="/sobrepaulo"> Sobre paulo

Trabalham com o recarregamento total de todas as páginas do projeto.
Ao utilizar a âncora, ele ignora totalmente as funcionalidades do REACT.

Link
< Link to="/sobrepaulo"> Sobre Paulo < /Link> <

Trabalha 100% com carregamento Nativo, carregando apenas os elementos necessários a cada atualização de página.

Prática

1) Construir o link para as páginas do seu projeto.
2) Conectá-los a sua página Home e aplicá-los no seu menu.
