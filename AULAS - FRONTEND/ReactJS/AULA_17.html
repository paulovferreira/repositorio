ASYNC / AWAIT

Esse conceito é puramente do JavaScript e é previsto exatamente para tratar processamentos paralelos dentro de aplicativos Web.
Não conseguimos garantir que um processo aguardará a conclusão de outro processo para que seja executado.

Por meio de um simples ALERT podemos verificar o funcionamento dos processos assincronos.
alert('Processo executado.')

Verifique o arquivo requisicoesTypesAsync para mais detalhes.

Trabalhando com LOAD.

Ao carregar a página é possível ver que existe um certo DELAY entre a requisição e o resultado da tela.
para tratar esses casos, precisaremos criar um LOADING para aperfeiçoar a interação com o usuário final do sistema.

Para isso, criaremos uma constante para sinalizar se o sistema está processando ou não a rotina externa.
const [loading, setLoading] = useState(false);

Consulte a utilização prática do Loading no arquivo requisicoesTypesAsync.tsx

Lidando com ERROS de carregamento

No momento do carregamento de nossas informações via API, podem ocorrer falhas que impossbilitem o carregamento da página e para isso, nosso código deverá estar preparado.

TRY CATCH

try {
Tente executar esse comando.
} catch (e) {
Se der falha, execute isso:
setLoading(false)
alert('Falha ao carregar informações, tente mais tarde.')
console.error(e);
}
