ESTRUTURAS DE PROJETO

PROJETO CRA X PROJETO Vite

CRA
Em um projeto CRA, observamos a pastar PUBLIC. - Armazena todos os arquivos de inicialização do projeto.
- Dentre eles o INDEX.HTML
- Basicamente aponta para uma única DIV onde será carregado todo seu site. Vite
Os arquivos de inicialização ficam na Raiz
- O arquivo index.html é mais "limpo" e além de carregar a div, já inicia o código JAVA Script;

O apontamento da DIV ROOT

CRA
- O primeiro arquivo a ser carregado é o index.tsx

Vite
- O primeiro arquivo a ser carregado é o main.tsx

Ambos tem a mesma função e possuem o mesmo objetivo: INICIAR UMA DIV DO MEU INDEX.

NESTE MOMENTO CONVÉM SABERMOS QUE:
- O Index do projeto está em lugares distintos.
- A aplicação de fato, o CÓDIGO FONTE, está na pasta SRC.
- Todos os arquivos externos (imagens etc...) devem ficar na pasta PUBLIC.

MEU PRIMEIRO COMPONENTE

Tudo em React, É COMPONENTE.
No react, componentes são traduzidor por FUNÇÕES que retornam algo gráfico.

Como já vimos, o arquivo principal (index ou main) , aponta para um arquivo que retornará oque deverá ser exibido na tela.
Se o MAIN importa algo, o APP, precisa exportar algo.

Praticando:

1 - Apague todo o conteúdo do arquivo APP.tsx (na pasta source)
2 - Vamos criar uma função para retornar um texto: "olá mundo"
Para isso:

function App() {
return(
< div>Olá Mundo!< /div>
);
}
export default App;

JSX

O Código digitado dentro do nosso APP.tsx é um código typescript ;
O código é identico a sintaxe do HTML, porém é uma especificação do HTML que retorna um componente

function App() {
return(
< div>Olá Mundo!< /div>
);
}
export default App;

Poderiamos também criar também utilizando JAVASCRIPT NORMAL.

import React, { createElement } from "react";
function App() {
return(
React.createElement('Div', null, 'MEU NOME NÃO É JHONNY.' )
/* ONDE OS PARAMETROS SÃO:
- Elemento
- Atributos
- Valor
*/
);
}
export default App;

A FACILIDADE DE TRANSFORMAR O JAVASCRIP EM UM PSEUDO HTML, SE CHAMA JSX.
Antes de exibir na tela, o código é convertido por um compilador. No typescript o BABEL

Outro exemplo:

import React, { createElement } from "react";
