ROUTERS

Para profissionalizar a navegação entre páginas no React, devemos ter claro a diferença entre PÁGINAS E COMPONENTES.

Páginas

São as páginas que deverão ser exibidas a cada interação do usuário com o menu.

Componentes

São os elementos que compoem as páginas. Uma página é formada de N componentes.

PARA NAVEGAR ENTRE AS PÁGINAS, UTILIZAMOS O ROUTERS

Router é uma biblioteca do React que permite criar ROTAS de navegação entre as páginas de um sistema web
garantindo que todas as interações entre o usuário e o sistema passem pelo seu controle.

Instalando o Routers.

NPM INSTALL REACT-ROUTER-DOM


Após instalar a biblioteca ROUTER é necessário importá-la no arquivo principal do seu projeto para garantir que o contexto seja plenamento estabelecido em todo o projeto.

No seu arquivo Index.tsx
import {BrowserRouter} from 'react-router-dom';

Essa importação garantirá que todo o contexto será processado dentro do BrowserRouter


A partir de agora, o < APP/> será executado dentro do BrowserRouter

< BrowserRouter>
< App />
< /BrowserRouter>

CRIANDO MULTIPLAS PÁGINAS

Para criarmos as rotas, é necessário apontar para quais páginas o destino fará referência.


1 - Criar Pasta "pages" dentro do "src" do seu projeto
2 - Criar arquivo Home.tsx e seu elemento interno Home (com Export)
3 - Criar arquivo Sobre.tsx e seu elemento interno Sobre (com Export)

CRIANDO ROTAS

A partir de agora, precisamos indicar no nosso arquivo principal (APP.TSX) quais as nossas rotas.
Esta ação tornará o site "mapeado" para que possa ser acessado via rotas pré-definidas.


NO APP.TSX
import { Routes, Route } from 'react-router-dom';
import Home from './pages/Home';
import Sobre from './pages/About';

NO RETURN DA FUNÇÃO
< Routes>
< Route path="/" element={< Home />} />
< Route path="/sobre" element={< Sobre />} />
< /Routes>

PRÁTICA

1 - Criar todas as páginas do seu projeto e referenciá-las por meio das rotas.
